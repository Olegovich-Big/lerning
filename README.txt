Каждое задание сделанно в отдельном branch:
1-3 лабороторная работа в branch - lb2-3 задание:
1 lab: 
    1) [5] Скачать и установить Node.js.
    2) [5] Установить TypeScript как зависимость проекта.
    4) [20] Presentation Maker.
    Необходимо прочитать требования к проекту Presentation Maker и описать следующие кастомные типы:  
        Презентация, 
        Слайд, 
        Коллекция слайдов, 
    Выделение.
    Также необходимо разработать функции по модификации данных: 
    изменение названия презентации
    добавление/удаление слайда
    изменение позиции слайда
2 lab:
    1) [5] Установить и настроить ESLint для TypeScript. https://typescript-eslint.io/getting-started/

    2) [10] Тестовые данные.
    Необходимо разработать тестовые данные и проверить работу функций, написанных в 1 лабораторной работе.
    Тестовые данные должны быть 2 типов:
    Минимальные данные. Если в модели есть опциональные поля, то этих полей не должно быть. Если в модели есть коллекции (массивы, мапы, множества и тд), то они должны быть пустыми.
    Максимальные данные. Должны быть все опциональные поля, во всех коллекциях должны быть минимум 2 элемента, должны быть все типы объектов.
    Каждая функция, написанная в 1 лабораторной работе должна вызываться дважды - с минимальными и максимальными данными. Код должен быть написать на TypeScript.
3 lab:
    1) [20] Необходимо сверстать интерфейс приложения, минимально должно быть:
    	рабочая область
        список слайдов
        название презентации
    Должна быть возможность отрендерить приложение с минимальными и максимальными тестовыми данными, разработанными во 2 лабораторной работе.

4-5 лабороторная работа в branch - lb4-5 задание:
4 lab:
    1) [20] Необходимо добавить стили к разработанным ранее компонентам. Стили должны быть написаны с использованием css модулей. Для применения динамических стилей можно использовать inline css.
5 lab:
    1) [20] Добавляем интерактив в наше приложение.
    Необходимо реализовать:
    создание/удаление слайдов
    добавление/удаление текста и картинок
    изменения фона слайда
    изменения названия презентации
При выполнении лабораторной работы необходимо использовать модуль editor.js, рассмотренный на лекции.

6 лабороторная работа в branch - lb6 задание:
    1) [5] Необходимо установить плагин в ESLint для проверки правил хуков.
    2) [20] В редакторе презентаций необходимо добавить:
        перемещение слайда (слайдов) drag and drop - ом;
        перемещение объекта (объектов) на слайде drag and drop - ом;
        изменение размеров (ресайз) объекта на слайде drag and drop-ом. Изменение размеров объекта должна происходить по 8 точкам.
    Функционал должен быть реализовать с помощью хуков в реакте.

7 лабороторная работа в branch - lb7 задание:
    1) [5] Сохранение документа в local storage. Все изменений, которые происходят в редакторе должны сохраняться в local storage, а при обновлении страницы необходимо получать сохраненное состояние из local storage.
    2) [10] Импорт и экспорт документа. Необходимо добавить функционал по импорту и экспорту документа в формате json (при желании  можно придумать свой формат). 
    3) [10] Валидация документа. Необходимо добавить валидацию документа сохраненного в local storage и на диск. Валидация должна происходить при загрузки документа из local storage или с диска (при импорте). При валидации должны проверяться существуют ли все обязательные поля и тип данных этих полей. 
    При выполнении 7.1 и 7.2 нельзя использовать сторонние библиотеки.
    Для выполнения задания 7.3 можно использовать библиотеку ajv: https://ajv.js.org/

8 лабороторная работа в branch - lb8 задание:
    [20] Подключите к вашему проекту state manager Redux. Все операции, изменяющие состояние приложение должны быть переведены в state manager.
    Рекомендуется использовать библиотеку react-redux для более удобной работы со state manager-ом.
    Redux toolkit использовать нельзя.
